協同產品設計實習 - 第七週回顧

一、課程背景

  • 協同產品設計實習課程：
  • 利用網際內容管理系統 (CMS)
  • 採用 Github Classroom 建立分組倉儲
  • 使用 Python 執行網頁編輯並轉檔為靜態網頁
  • 利用 Github Pages 呈現協同產品設計過程的多媒體內容

二、內容管理系統 (CMS)

  • 主要工具：
  • cmsimde_site 作為 Github 倉儲的引擎目錄
  • Python 執行網頁編輯
  • Pelican 管理網誌
  • Reveal 管理網際簡報

三、網頁編輯方法

  • 四種方法：
  • Replit
  • Codespaces
  • Gitpod
  • Localhost

四、協同模式

  1. 分組網站內容的協同：
  2. 章節式網站內容
  3. 網誌內容 (markdown 轉為超文件)
  4. 網際簡報內容
  5. 分組報告內容的協同：
  6. 使用 Github Actions 與 MikTeX 進行 LaTeX 轉 PDF
  7. 分組產品設計的 PLM 協同：
  8. 使用 Odoo PLM 模組管理設計過程
  9. 分組零組件設計的協同：
  10. 使用 Solvespace 和 Siemens NX 進行零組件設計
  11. 分組模擬場景設計的協同：
  12. 使用 CoppeliaSim 進行模擬
  13. 分組專案簡報設計的協同：
  14. 使用 Reveal 進行簡報設計

五、工具與程式

  • Python 程式工具：
  • add_to_content_html.py
  • generate_pages.py
  • Bash Script：
  • um (update submodule)

六、協同過程詳解

  1. 分組網站內容的協同：
  2. 每週編寫 markdown 網誌
  3. 使用 Pelican 轉檔至 blog 目錄
  4. 分組報告內容的協同：
  5. 使用 LaTeX 格式完成報告
  6. 結合 Odoo PLM 管理協同過程
  7. 分組零組件設計的協同：
  8. 使用 Siemens NX 繪製零件並錄製 NXOpen Python Journal 程式
  9. 分組模擬場景設計的協同：
  10. 使用 ZMQ Remote API Python 控制模擬場景
  11. 分組專案簡報設計的協同：
  12. 利用 Markdown 格式內容進行簡報設計

七、導入 AI 的流程

  • 使用工具：
  • ChatGPT 3.5
  • Gemini Pro
  • Claude 3
  • 應用範例：
  • 將 Odoo 參考論文上傳進行翻譯與摘要提取
  • 利用 AI 工具回答設計與程式相關問題

八、Odoo 參考論文摘要

  1. 緒論：
  2. 探討 Odoo 在 PLM 和 MES 整合方面的能力
  3. 理論基礎：
  4. PLM、ERP 和 MES 的定義與重點
  5. PLM 和 MES 的整合：
  6. 整合目標與方法
  7. 公司和產品介紹：
  8. 案例公司與產品
  9. Odoo 軟體介紹：
  10. Odoo 概述與製造功能
  11. Odoo 在 PLM 和 MES 方面的表現：
  12. 專案管理、產品和制程創建、產品和制程改進、數據查詢
  13. 結論：
  14. Odoo 作為小型企業 PLM 和 MES 解決方案的優劣勢
