第四週協同產品設計實習詳細大綱

一、概述

  • 目標：明確界定各分組的任務需求與分工時序，使用 SourceTree 和 Git Branch 優化協同設計流程。
  • 工具：SourceTree、Git Branch、GitHub、Replit、Codespaces、Gitpod、localhost。

二、工具與方法介紹

  1. SourceTree
  2. 用途：視覺化管理 Git 倉儲，清晰查看改版歷程。

  3. 操作：如何安裝、連接到遠端倉儲，基本操作。

  4. Git Branch

  5. 用途：減少合併衝突，管理階段性改版內容。
  6. 操作：創建、切換、合併分支的具體步驟。

三、任務需求與分工時序

  1. 界定需求

  2. 每個分組需明確界定任務需求，包括教學影片字幕、逐字稿、摘要說明等。

  3. 分工時序

  4. 各分組成員的具體分工及工作進度安排。

四、教學影片整理與自評

  1. 任務
  2. 為教學影片添加字幕。
  3. 整理出逐字稿並放置在影片下方，以 .txt 連結方式安排。

  4. 編寫教學影片的摘要。

  5. 自評

  6. 分組內部進行自評，確保任務完成質量。

五、GitHub 個人與分組倉儲維護

  1. 倉儲結構
  2. 個人倉儲：cd2024

  3. 分組倉儲：mdecd2024，命名規則 2a-midag<組別>。

  4. 子模組設置

  5. 將個人倉儲設為分組倉儲的子模組，便於各組員自行管理和維護。

六、資料管理

  • 類型：
  • 文字資料（含 LaTeX 報告章節）
  • 零組件檔案、工程圖、組立圖（含 BOM 與爆炸圖）
  • 教學影片、操作流程錄製影片或 Wink 資料。

七、倉儲維護方式介紹

  1. Replit
  2. 歷史及用途介紹。
  3. 免費帳號限制及使用方法。

  4. 動態與靜態網站維護的具體步驟及注意事項。

  5. Codespaces

  6. 使用方法及限制介紹。

  7. 動態與靜態網站維護的具體操作步驟。

  8. Gitpod

  9. 使用方法及免費授權時間限制。

  10. 動態與靜態網站維護的具體操作步驟。

  11. localhost

  12. 使用可攜程式系統維護倉儲。
  13. 動態與靜態網站維護的具體操作步驟。

八、NX 繪圖協同設計

  1. 軟體版本
  2. 各版本 NX 的安裝與啟動方法。

  3. USB 隨身碟啟動批次檔案配置。

  4. 協同設計

  5. 不同版本 NX 的協同設計方法及注意事項。

九、討論與反思

  1. 留言系統

  2. 使用留言系統進行討論，交流實習過程中的問題和經驗。

  3. 反思

  4. 根據反思一和反思二，理解協同設計的重要性和團隊合作的必要性。
